    취준때 시작한 티스토리 블로그를 약2년이 지난 지금 블로그를 마이그레이션 하려 하고 있어요.

    티스토리를 이용하며 겪은 불편함이 있었는데요.

    제가 겪은 불편함은 블로그 제목을 바꾸는것에 대한 제한과 디자인을 바꾸기가 쉽지가 않다는 것이었어요.

    그래서 이번 기회에 블로그를 직접 만들어보며 css연습과 프레임워크 공부를 함께 해 두 마리 토끼를 잡으려 했습니다. 👍


    대표적으로 사용되는 우리나라 개발 블로그는 tistory, velog, github page 일 것 같은데요.

    저는 모든걸 직접 만들 수 있는 github page를 선택 했어요.

    이번 블로그에 사용되는 대표적인 스택은 이렇게 3가지 인데요.

  1. next.js 14v
  2. tailwind css
  3. ts


    4. github page에 next를 셋팅하는게 쉽게 되지는 않더라구요.

    js 파일이 load가 안되어 기능이 작동을 안한다거나.. tailwind 적용이 안된다거나.. 🫠

    어쨋든 문제점들은 잘 해결되었고, 이제 약 한달 정도 마이그레이션 진행중이네요.

    작업량이 많지는 않지만, tailwind를 잘 사용할 수 있을까에 대한 걱정이 있었는데, 원하는데로 디자인이 잘 적용되고 있어

    나름 만족하며 진행중인 것 같네요. 😄


    한가지 더 알려드릴 내용으로, 개인적인 욕심 때문에 notion db도 사용하고있어요.

    사실 notion에 작성한 글이 블로그에도 자동으로 작성이 되면 좋겠다는 생각을 했었거든요.

    notion db를 사용하면, notion에도 글을 작성 후 빌드를 하면, 블로그에 글이 자동으로 포스팅 될거에요


    다음 회고때는 아웃풋이 어느정도 나온 뒤 작성이 될 것 같네요 ✋
